## Задача

Ко мне обратился клиент с задачей автоматизировать работу с телефонией Zadarma. Каждый день ему приходилось вручную заходить в панель, искать записи звонков за последние 24 часа и выгружать аудиофайлы для дальнейшей обработки. Это занимало много времени и было подвержено ошибкам.

Нужно было создать n8n workflow, который автоматически будет подключаться к API Zadarma, вытягивать аудиофайлы за последние 24 часа и передавать их дальше в обработку. Результатом должен был стать готовый workflow, который можно скопировать в любую n8n-инсталляцию с возможностью настроить собственные API-реквизиты.

## Решение

Я создал полноценную кастомную n8n-ноду с нуля для интеграции с Zadarma API. Реализовал надёжную аутентификацию на основе рабочей Python-реализации, что обеспечило стабильное подключение к API. Затем построил workflow, который заходит с указанными пользователем API-данными в историю записей Zadarma, фильтрует записи за последние 24 часа и по очереди выдаёт аудиофайлы для дальнейшей обработки.

Добавил retry-механику для неуспешных загрузок, статистику скачивания и настраиваемый формат имён файлов. Также сделал гибкую систему конфигурации через переменные окружения, включая API-ключи, sandbox-режим, количество дней для поиска и папку для сохранения файлов.

Отдельное внимание уделил документации и простоте использования. Подготовил подробный README с примерами использования, пояснениями по API и пошаговой инструкцией по настройке.

## Результат

В итоге получилась полнофункциональная n8n-нода, опубликованная в npm как `n8n-nodes-zadarma`. Клиент получил готовый workflow, который можно легко интегрировать в любую n8n-инсталляцию. Время на обработку аудиозаписей сократилось с часов ручной работы до автоматического фонового процесса.

Проект включает как Python-скрипт для локального использования, так и полноценную n8n-ноду для интеграции в рабочие процессы. Также я добавил поддержку GitHub Packages для более простого распространения и обновлений.
