## Завдання

До мене звернувся клієнт із задачею автоматизувати роботу з телефонією Zadarma. Щодня йому доводилося вручну заходити в панель, шукати записи дзвінків за останні 24 години та вивантажувати аудіофайли для подальшої обробки. Це займало багато часу і було схильне до помилок.

Потрібно було створити n8n workflow, який автоматично буде підключатися до API Zadarma, витягувати аудіофайли за останні 24 години і передавати їх далі в обробку. Результатом мав стати готовий workflow, який можна скопіювати в будь-яку n8n-інсталяцію з можливістю налаштувати власні API-реквізити.

## Рішення

Я створив повноцінну кастомну n8n-ноду з нуля для інтеграції з Zadarma API. Реалізував надійну аутентифікацію на основі робочої Python-реалізації, що забезпечило стабільне підключення до API. Потім побудував workflow, який заходить із заданими користувачем API-даними в історію записів Zadarma, фільтрує записи за останні 24 години і по черзі видає аудіофайли для подальшої обробки.

Додав retry-механізм для невдалих завантажень, статистику скачування і налаштовуваний формат назв файлів. Також зробив гнучку систему конфігурації через змінні середовища, включно з API-ключами, sandbox-режимом, кількістю днів для пошуку та папкою для збереження файлів.

Окрему увагу приділив документації та простоті використання. Підготував детальний README з прикладами використання, поясненнями по API та покроковою інструкцією з налаштування.

## Результат

У підсумку вийшла повнофункціональна n8n-нода, опублікована в npm як `n8n-nodes-zadarma`. Клієнт отримав готовий workflow, який можна легко інтегрувати в будь-яку n8n-інсталяцію. Час на обробку аудіозаписів скоротився з годин ручної роботи до автоматичного фонового процесу.

Проєкт включає як Python-скрипт для локального використання, так і повноцінну n8n-ноду для інтеграції в робочі процеси. Також я додав підтримку GitHub Packages для простішого розповсюдження та оновлень.
