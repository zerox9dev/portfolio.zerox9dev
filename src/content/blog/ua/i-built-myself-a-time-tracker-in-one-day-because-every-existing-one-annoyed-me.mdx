Я фрилансер. Я трекаю час щодня. За останній рік я спробував Toggl, Clockify, Harvest і кілька таблиць у Notion. Щоразу одна й та сама історія: або дорого за базові функції, або безкоштовно, але UI відчувається як пульт керування атомною електростанцією.

Toggl хоче $13/міс за нормальні фільтри. Clockify безкоштовний, але щоб дістатися до таймера, треба зробити п’ять кліків. Harvest — $11 за таймер. Просто за таймер.

Мені потрібно лише просте: натиснув Space — працюєш. Натиснув ще раз — зупинився. Наприкінці місяця — інвойс. І все.

## Накодив вайбом за день

Я дизайнер і сам коджу свої проєкти. У якийсь момент я подумав — чому я плачу за те, що можу зібрати собі за вечір? Ну, вийшов не вечір, а цілий день, але все одно.

Стек доволі стандартний: Next.js 16, React 19, Supabase для auth і бази даних, Vercel для деплою. Базу зробив максимально простою — одна таблиця `user_app_state` з JSONB для клієнтів і сесій. Не нормалізовано, так. Для MVP — ідеально. Менше міграцій, менше болю.

Зранку: накидав структуру. Вдень: зібрав трекер. Усе keyboard-first — Space для старту/стопу, Enter щоб додати клієнта, Escape щоб закрити. Жодних модалок там, де можна без них. Увечері: підключив Google Auth через Supabase, зробив лендинг, задеплоїв. Уночі: дополірував експорт інвойсів і CSV.

## Поточний стан

Він не ідеальний. Іноді я натрапляю на баги. Але весь свій час я вже трекаю тільки через нього — і щоразу, коли щось ламається, я одразу це виправляю. Справжній dogfooding у реальному часі.

## Що він уміє

- Таймер на Space
- Clients -> Projects -> Tasks
- Статуси: PENDING -> ACTIVE -> DONE
- Ручне додавання сесій (бо іноді забуваєш натиснути старт)
- PDF invoice в один клік
- Експорт у CSV
- Google auth + хмарна синхронізація
- fallback на localStorage — працює офлайн
- Темна/світла тема
